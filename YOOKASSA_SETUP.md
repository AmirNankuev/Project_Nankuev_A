# Тестовая оплата через ЮKassa

В проект добавлена тестовая интеграция ЮKassa для этапа оплаты после выбора доставки.

## Как работает сценарий

1. Покупатель заполняет данные на `/checkout/`.
2. На `/checkout/delivery/` рассчитывает доставку СДЭК и выбирает способ оплаты ЮKassa.
3. После подтверждения заказа система создаёт локальный заказ.
4. Для онлайн-оплаты создаётся платёж ЮKassa.
5. Покупатель переходит на страницу оплаты.
6. После возврата на сайт открывается `/checkout/payment/<номер-заказа>/`, где система проверяет статус платежа.

## Демо-режим по умолчанию

По умолчанию включён локальный демо-режим:

```python
YOOKASSA_DEMO_MODE = True
```

В этом режиме реальные запросы в ЮKassa не отправляются. На странице оплаты появляется кнопка «Оплатить тестово». Она переводит заказ в статус «Оплачен».

## Подключение тестовой ЮKassa

Для проверки через настоящий тестовый контур ЮKassa отключите демо-режим и задайте ключи:

```bash
YOOKASSA_DEMO_MODE=false
YOOKASSA_SHOP_ID=ваш_тестовый_shop_id
YOOKASSA_SECRET_KEY=ваш_тестовый_secret_key
YOOKASSA_RETURN_BASE_URL=http://127.0.0.1:8000
```

Для локального запуска выполните:

```bash
python manage.py migrate
python manage.py runserver
```

## Файлы интеграции

- `main/services/yookassa_payment.py` — клиент ЮKassa API.
- `catalog/views.py` — создание платежа, редирект на оплату, проверка статуса.
- `templates/pages/checkout_payment.html` — страница оплаты.
- `main/models.py` — поля заказа для хранения ID платежа, статуса, ссылки и ответа API.
- `main/migrations/0003_order_yookassa_fields.py` — миграция полей ЮKassa.
- `main/admin.py` — отображение данных ЮKassa в админке.

## Что сохраняется в заказе

- `yookassa_payment_id` — ID платежа ЮKassa.
- `yookassa_payment_status` — статус платежа.
- `yookassa_confirmation_url` — ссылка на страницу оплаты.
- `yookassa_response` — полный ответ API.
- `yookassa_error` — текст ошибки, если платёж не создался или статус не проверился.

## Примечание

Для учебного проекта не реализована фискализация чеков и webhook-уведомления. Статус платежа проверяется при открытии страницы возврата `/checkout/payment/<номер-заказа>/` или по кнопке «Проверить статус».
