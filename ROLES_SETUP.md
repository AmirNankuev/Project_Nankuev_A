# Роли пользователей

В проект добавлена модель ролей для `users.CustomUser`:

- **Покупатель** (`customer`) — обычный пользователь интернет-магазина. Может пользоваться каталогом, корзиной, оформлять заказы, смотреть свои заказы и создавать заявки на возврат.
- **Менеджер** (`manager`) — сотрудник магазина. Имеет доступ к рабочей панели и Django Admin для обработки каталога, заказов, возвратов и складских остатков. Не управляет пользователями и не имеет права полного администратора.
- **Администратор** (`administrator`) — полный доступ к системе, включая управление пользователями и ролями.

## Как применить изменения

После обновления кода выполните миграции:

```bash
python manage.py migrate
```

Миграция добавит поле `role` в таблицу пользователей и автоматически распределит существующих пользователей:

- `is_superuser=True` → Администратор;
- `is_staff=True` → Менеджер;
- остальные → Покупатель.

Для повторной ручной синхронизации групп и прав можно выполнить:

```bash
python manage.py setup_roles
```

## Где менять роль

Роль меняется в админке:

`/admin/` → **Пользователи и роли** → **Пользователи** → поле **Роль в системе**.

После сохранения автоматически синхронизируются:

- `is_staff`;
- `is_superuser`;
- группа Django (`Покупатели`, `Менеджеры`, `Администраторы`).

## Важные файлы

- `users/models.py` — поле `role` и свойства ролей.
- `users/roles.py` — группы, права и синхронизация.
- `users/signals.py` — автоматическая синхронизация роли после сохранения пользователя.
- `users/decorators.py` — ограничения доступа для покупателя и менеджера.
- `users/views.py` — личный кабинет покупателя и рабочая панель сотрудника.
- `templates/registration/staff_dashboard.html` — панель менеджера/администратора.
- `users/migrations/0002_customuser_role.py` — миграция роли.
